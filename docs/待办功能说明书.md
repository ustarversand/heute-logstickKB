# 📋 待办事项 — 功能说明书

## 功能概述

在货易达物流看板（port 8890）的 📋 待办 Tab 中，展示**未上传身份证正反面**的订单，支持按寄件人隔离、一键上传身份证到认证系统。

## 后端 API

### 1. GET /api/todo — 拉取待办列表

```python
@app.get('/api/todo')
def api_todo(request: Request, month: str = None):
```

**流程：**
1. 调清关看板 `http://192.168.178.26:18995/api/missing/recent` 获取未认证身份证清单
2. 按 `sender_name`（寄件人）分组
3. 调用 `get_sender_from_session(request)` 获取当前登录用户的寄件人
4. **非管理员**：过滤只保留自己的寄件人数据

**返回格式：**
```json
{
  "items": [...],
  "senders": {
    "xiaoxiaPDD": [
      {"consignee": "张三", "id_number": "440...", "domestic_no": "JDV...", 
       "line_name": "中欧专线", "sn": "SN123", "global_waybill": "1Z..."}
    ]
  },
  "total": 20,
  "sender_count": 8
}
```

**隔离逻辑 (`get_sender_from_session`)**：
```python
def get_sender_from_session(request):
    role = request.session.get('role', '')
    # admin = 全部可见，其他人 = 只看自己的寄件人
    return None if role == 'admin' else request.session.get('sender', None)
```

### 2. POST /api/todo/upload — 上传身份证

```python
@app.post('/api/todo/upload')
def api_todo_upload(request: Request,
                    name: str = Form(...),
                    id_number: str = Form(...),
                    front_image: UploadFile = File(...),
                    reverse_image: UploadFile = File(...)):
```

**流程：**
1. 接收 multipart 表单（name + id_number + 正反面图片文件）
2. 限制单张 ≤ 10MB
3. 调用 `idcard_upload.upload()` 中转至直邮管家
4. 返回成功/失败

## 上传中转模块 (`idcard_upload.py`)

核心函数：

```python
def upload(name: str, id_number: str, 
           front_data: bytes, reverse_data: bytes) -> dict:
```

**步骤：**
1. **登录直邮管家**：`POST http://192.168.178.26:8899/api/login` 获取 session cookie
2. **构建 multipart 请求**：手动拼接 MIME 边界（无 `requests` 库）
3. **转发上传**：`POST http://192.168.178.26:8899/api/idcard/upload` + session cookie
4. 直邮管家内部处理：保存图片 → OCR 识别 → HMAC 签名 → 调用 CCS API

**认证信息**：直邮管家 admin 密码通过环境变量注入
```python
ZYGJ_URL = "http://192.168.178.26:8899"
ZYGJ_ADMIN = "admin"
ZYGJ_PASS = "Hilden11031980"
```

## 数据来源说明

待办列表的数据来自 **清关看板**（customs-clearance-dashboard，port 18995），不是货易达物流看板自己的数据库。

清关看板的 `/api/missing/recent` 返回**近60天内活跃、未撤销、缺身份证认证的订单**。这份数据由清关看板的同步脚本定期从货易达 API 拉取。

## 部署注意事项

1. **上传依赖直邮管家在线** — 如果 zhiyouguanjia 容器挂了，上传会报连接失败
2. **图片大小限制** — 单张 ≤ 10MB，格式支持 JPEG/PNG
3. **隔离仅限看板** — 寄件人隔离是看板登录层面的，清关看板的数据本身包含全部寄件人
4. **数据库持久化** — 修改 Dockerfile 后必须重新 build 镜像，`./data` 目录通过 volume 持久化
